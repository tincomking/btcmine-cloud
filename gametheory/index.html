一、定价模型架构

折旧基础定价 vs 传统 Hashprice 定价

传统云算力平台以 hashprice 为基础定价：卖价 = hashprice × (1 + premium)。这在数学上保证用户成本 > 挖矿收入（ROI 永远为负）。

本模型采用折旧基础定价：将硬件使用权价格锚定在折旧成本上。折旧远小于 hashprice（$0.013 vs $0.045），为三方利润留出空间。

核心公式

// Hashprice: 每 TH/s 每天的美元收入 daily_btc_per_th = (144 × 3.125) / (network_EH × 1,000,000) hashprice = daily_btc_per_th × btc_price // 矿工电费成本 elec_cost_per_th = efficiency_J_TH × 24 / 1000 × elec_rate [$/TH/天] // 三层定价 hw_rental = depreciation × (1 + premium) ← 矿工收入 user_hp = hw_rental / (1 - commission) ← 用户算力费 user_elec = actual_elec × (1 + markup) ← 用户电费 user_cost = user_hp + user_elec ← 用户总日成本 // 用户 ROI ROI = (hashprice - user_cost) / user_cost

Premium 调整机制

effective_premium = base + period_adjust + volume_discount 最终值 = max(10%, effective_premium) 周期调整: 30d → +15% 90d → 0% 120d → -5% 180d → -10% 360d → -20% 量折扣: <500 → 0% 500+ → -3% 1000+ → -5% 5000+ → -8% 示例: base=70%, period=180d, vol=5000 TH → 70% + (-10%) + (-8%) = 52%

市场三阶段

A 震荡上行（Epoch 1-12, 2025.03—2025.08） — BTC 从 $81K 涨至 $117K，全网算力从 846 EH 增至 1,180 EH（+39%），持续触达 +4% 增长上限。BTC 涨幅被算力增长部分抵消，hashprice 在 $0.044-$0.051 区间波动。匹配量稳定在 10,200-12,100 TH/epoch。

B 高位转下行（Epoch 13-24, 2025.08—2026.01） — BTC 从 $118K 跌至 $89K，算力惯性增长后回落（1,381 → 1,203 EH）。价格下跌 + 算力高位 → hashprice 持续压缩至 $0.033。平台被迫降费率，但匹配量反升至 16K（更多矿工放弃自挖）。

C 深度回调（Epoch 25-26, 2026.01—2026.02） — BTC 暴跌至 $67-71K，hashprice 跌破 $0.028。用户成本 > 挖矿收入，所有买家退出。全部矿工愿意卖（自挖亏损更大），但无人接盘。市场冻结。

四、买家决策分析

决策逻辑

// 用户 ROI（当前价格，不含升值预期） user_daily_cost = user_hp + user_elec total_cost = user_daily_cost × buy_TH × period total_btc = daily_btc_per_th × buy_TH × period ROI = (total_btc × btc_price - total_cost) / total_cost // 挖矿 vs 买币 比值 mining_vs_buying = total_btc / (total_cost / btc_price) // 决策：ROI ≥ eff_min_roi AND mvb ≥ eff_mvb → 买算力 // 有效阈值受趋势、波动率、后悔学习调整

Hashprice 与 ROI 的非线性关系

反直觉现象：BTC 价格最高时（$118K, Epoch 16），买家 ROI 仅 +5.4%，因为全网算力大增压低了 hashprice。Epoch 19 BTC 跌到 $96K，但算力同步回调、平台大幅降费，ROI 反升至 +9.7%。

买家退出阈值：hashprice ≈ $0.030 时用户成本开始超过挖矿收入（ROI 转负），对应 BTC ≈ $75K。
